# 平面设计师方案产出指南

## Overview

本技能将平面设计领域的方法论转化为可执行的工作流。当用户提出任何视觉设计需求时，先识别该需求属于 5 类场景中的哪一类，再按对应场景的产出清单生成完整的设计方案和交付规格——从品牌策略到物料输出，覆盖品牌设计的完整生命周期。

详细的方法论、各场景产出清单、色彩/分辨率/出血规范、版权检查清单、物料命名规范、AI工具建议、质量检查清单均存放在 `references/平面设计方法论.md`，在执行前必须读取对应章节。

## 触发条件

当出现以下任一情况时触发本技能：

- 用户需要设计新品牌VI或品牌视觉系统
- 用户需要设计营销活动物料（海报/Banner/社交媒体素材等）
- 用户需要设计单件物料（名片/Logo/海报/折页）
- 用户需要做品牌焕新/VI升级
- 用户需要风格探索或 Mood Board
- 用户提到"Logo""VI""品牌色""海报""Banner""视觉设计""平面设计""物料"等关键词

## 执行流程

按以下 7 步顺序执行，不可跳步。

### Step 0: 加载历史记忆

在执行本技能前，先检查并加载历史记忆：

1. 检查项目目录下是否存在 `.skill-memory/graphic-design-guide/` 目录，如不存在则创建
2. 读取 `.skill-memory/graphic-design-guide/MEMORY.md`（如果存在）——了解长期品牌规范、色彩体系、字体授权清单、视觉风格约定等
3. 读取 `.skill-memory/graphic-design-guide/YYYY-MM-DD.md`（今日日志，如果存在）——了解近期已完成的设计任务

加载记忆后，在回复中简要提及相关的历史上下文（如"根据之前定义的品牌规范，主色为 #C8923F..."），让用户感受到连贯性。

### Step 1: 需求理解

- 解析用户输入的视觉设计需求
- 提取关键信息：设计载体（印刷/屏幕/两者）、品牌现有VI情况、目标受众、使用场景、风格偏好、是否有品牌约束
- 识别缺失的关键信息，主动向用户提问补全（一次最多 2-3 个最关键的问题）

### Step 2: 场景识别

读取 `references/平面设计方法论.md` 的"一、场景识别"章节，用下表判断：

| 场景 | 名称 | 判断条件（满足任一） | 产出量 |
|------|------|---------------------|--------|
| 场景一 | 0→1 品牌识别设计 | 新品牌/子品牌、无现有VI、需定义品牌视觉语言 | 10-12类 |
| 场景二 | 营销活动物料 | 主题活动/节日/促销、多媒介(线上+线下)、多尺寸 | 6-8类 |
| 场景三 | 单件物料制作 | 单张海报/单个Banner/单个图标 | 2-3类 |
| 场景四 | 品牌焕新/VI升级 | 品牌重塑、LOGO/色彩/字体变更、涉及全渠道切换 | 8-10类 |
| 场景五 | 风格探索/Mood Board | 方向探索、灵感收集、风格提案 | 3-4类 |

识别流程：是否新品牌/无VI？→ 场景一；是否品牌焕新/VI升级？→ 场景四；是否为风格探索？→ 场景五；是否涉及多媒介/多尺寸/系列活动？→ 场景二；否则 → 场景三。

### Step 3: 与用户确认场景

输出场景判断结果，包含：
1. 场景判断：场景X
2. 判断依据
3. 建议产出清单
4. 预估周期

向用户确认后进入产出阶段。

### Step 4: 按清单产出设计方案

读取 `references/平面设计方法论.md` 中对应场景的章节，按场景获取必备产出清单。

产出要求：
- 设计方案的输出为**详细文字描述 + 规格参数**（色彩值、尺寸、字体、材质、工艺），便于直接执行
- 色彩标注必须同时给出 HEX + CMYK + PANTONE（如涉及印刷）
- 字体选择必须注明授权情况（免费商用/需购买/已有授权）
- 印刷物料必须标注：纸张类型/克重、工艺（覆膜/烫金/UV等）、出血3mm
- 数字物料必须标注：各平台规范尺寸、文件格式、色彩模式
- 遵循"七、通用交付标准"中的规范
- 产出后保存为 Markdown 文件

### Step 5: 质量检查

读取 `references/平面设计方法论.md` 的"九、产出质量检查清单"章节，对照检查：

**品牌一致性：** 色彩/字体/LOGO使用正确、品牌调性一致
**技术规范：** 色彩模式正确、分辨率达标、出血安全区正确、字体转曲
**版权合规：** 图片版权确认、字体商用授权确认、肖像权确认
**交付完整：** 源文件 + 导出文件 + 字体文件 + 交付清单

识别遗漏内容并补充，最后输出交付清单。

### Step 6: 记录执行记忆

完成实质性设计工作后，将关键信息持久化到项目记忆目录：

1. **写入今日日志** — 向 `.skill-memory/graphic-design-guide/YYYY-MM-DD.md` **追加**（不存在则先创建）：
   - 场景类型（场景X）
   - 设计的品牌/物料名称
   - 产出的设计方案清单
   - 关键设计决策（如品牌色值、字体选择、LOGO变体方案、物料规格等）
   - 待后续跟进的事项

2. **更新长期记忆** — 如果产生了**可复用的品牌规范或设计决策**（如品牌VI规范、色彩体系、字体授权清单、物料模板等），同步更新 `.skill-memory/graphic-design-guide/MEMORY.md`：
   - 按主题分类（品牌VI / 色彩体系 / 字体方案 / 物料规范 / 版权清单）
   - 保持简洁，每条 1-2 行

**只需记录实质性工作**（产出了品牌方案/VI规范/活动物料方案等）。纯问答、风格探索不需记录。

3. **同步到仓库** — 可将 `.skill-memory/graphic-design-guide/` 下的变更提交到 Git，实现团队共享记忆。

**记忆目录结构：**
```
项目目录/.skill-memory/graphic-design-guide/
├── 2026-07-22.md
└── MEMORY.md
```

## 资源说明

### references/平面设计方法论.md

完整的方法论文档，包含：
- 5个场景的详细产出清单
- 色彩模式规范（RGB/CMYK/PANTONE）
- 分辨率规范（屏幕72PPI/印刷300DPI/高清印刷350+DPI/户外72-150DPI）
- 出血与安全区规范
- 图片版权检查清单
- 源文件组织与命名规范
- AI 工具使用建议
- 产出质量检查清单（品牌一致性/技术规范/版权合规/交付完整）

## 注意事项

- 不要跳过 Step 3 的用户确认
- 设计产出以**文字描述 + 规格参数**为主，不是直接出图
- 色彩标注必须多模式给全（HEX + CMYK + PANTONE 如涉及印刷）
- 版权是硬性要求——字体和图片的商用授权不能省略
- 印刷物料的出血（3mm）和安全区（距边5mm）是硬规范
- 场景四（品牌焕新）的切换计划和过渡期双标管理是硬性要求
